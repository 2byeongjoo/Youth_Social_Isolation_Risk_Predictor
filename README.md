## 1. 소개
청년의 사회적 고립 위험도를 예측하는 머신러닝 기반 웹입니다.

본 프로젝트는 서울시 고립은둔청년 실태조사 데이터를 활용하여, 사용자의 외출 수준, 사회적 교류 빈도, 외로움 관련 응답 등을 입력값으로 받아 고립은둔 가능성을 예측합니다.

모델은 로지스틱 회귀 기반의 이진 분류 모델을 사용하였으며, 웹 화면에서는 예측 확률을 바탕으로 고립 위험도를 미해당/고립은둔 청년 형태로 제공합니다.

본 서비스는 의학적·심리적 진단 도구가 아니라, 공공데이터 기반 학습용 예측 웹입니다.

data/processed/isolation_selected.csv

models/logistic_model.pkl

src/youth_isolation/
머신러닝 관련 코드가 들어가는 폴더입니다.

## 3. <> 사용 데이터
본 프로젝트는 서울시 고립은둔청년 실태조사 청년조사 데이터를 사용했습니다.

사용한 원본 데이터는 엑셀 파일 형태이며, Data 시트에는 실제 응답 데이터가 들어 있고, GUIDE 시트에는 각 컬럼의 의미와 응답값 설명이 정리되어 있습니다.

본 프로젝트에서는 전체 설문 문항 중 사회적 고립 위험도 예측과 관련성이 높다고 판단한 외출 수준, 사회적 교류 빈도, 온라인 대화 여부, 외로움 관련 문항 등을 선택하여 모델 학습에 사용했습니다.

원본 데이터 파일은 용량 및 관리 문제로 GitHub에는 포함하지 않고, 로컬 환경의 data/raw/ 폴더에 별도로 배치합니다.

원본 데이터 파일 위치는 다음과 같습니다.

data/raw/youth_isolation.xlsx

전처리 후 생성되는 데이터 파일 위치는 다음과 같습니다.

data/processed/isolation_selected.csv

전처리 과정에서는 원본 엑셀 파일의 Data 시트를 불러온 뒤, 모델 학습에 필요한 입력 변수와 정답 라벨 컬럼만 선택했습니다. 이후 결측치를 확인하고, 결측치가 포함된 행을 제거한 뒤 전처리 결과를 CSV 파일로 저장했습니다.

선택한 컬럼 기준 원본 데이터는 5,513건이었으며, 결측치 제거 후 5,354건이 모델 학습에 사용되었습니다.

## 4. <> 입력 변수
모델 학습에 사용한 입력 변수는 다음과 같습니다.

컬럼명	의미
A7	평소 외출 수준
A10	지난 2주 동안 만나 이야기 나눈 사람 수
A11	지난 2주 동안 누군가와 대화하거나 함께 시간을 보낸 횟수
A13_1	가족들과의 대면 교류 빈도
A13_3	친한 친구나 친한 사람과의 대면 교류 빈도
A13_4	직장, 학교, 동네 사람들과의 대면 교류 빈도
A17	온라인 커뮤니티, SNS, 메신저 등에서 대화하는지 여부
A18_1	나와 같이 있어 줄 사람이 부족하다고 느끼는 정도
A18_3	혼자 남겨진 것 같다고 느끼는 정도
B1_2	평소 가족과 대화를 자주 하는 편인지

A7은 단순한 외출 횟수 값이 아니라 설문 응답 코드입니다. 값이 작을수록 외출이 많은 상태에 가깝고, 값이 클수록 외출이 제한적인 상태에 가깝습니다.

본 프로젝트에서는 사용자의 사회적 고립 위험도와 관련될 수 있는 생활 패턴, 대면 교류, 온라인 교류, 외로움 관련 응답을 중심으로 입력 변수를 선정했습니다.

## 5. <> 정답 라벨
모델의 정답 라벨은 KEY_1 컬럼을 사용했습니다.

KEY_1은 고립은둔 여부를 나타내는 컬럼입니다.

값	의미
1	고립은둔 청년
2	미해당

따라서 본 프로젝트는 사용자의 입력값을 바탕으로 고립은둔 여부를 예측하는 이진 분류 문제로 정의했습니다.

결측치 제거 후 정답 라벨 분포는 다음과 같습니다.

라벨	의미	개수
1	고립은둔 청년	369
2	미해당	4,985

고립은둔 청년에 해당하는 데이터가 상대적으로 적기 때문에 클래스 불균형이 존재합니다. 이에 따라 모델 학습 시 class_weight="balanced" 옵션을 적용하여 소수 클래스가 지나치게 무시되지 않도록 했습니다.

## 6. <> 모델 설명
본 프로젝트에서는 1차 모델로 로지스틱 회귀 모델을 사용했습니다.

LogisticRegression(max_iter=1000, class_weight="balanced")

로지스틱 회귀를 사용한 이유는 다음과 같습니다.

정답 라벨인 KEY_1이 고립은둔 청년과 미해당을 구분하는 이진 분류 구조이기 때문입니다.
모델 구조가 비교적 단순하여 학습 과정과 결과를 설명하기 쉽습니다.
predict_proba()를 활용해 고립은둔 가능성 확률을 계산할 수 있습니다.
계산된 확률을 바탕으로 웹 화면에서 위험도를 낮음 / 보통 / 높음 형태로 변환할 수 있습니다.

모델 학습 과정은 다음 순서로 진행했습니다.

1. 전처리된 CSV 파일을 불러옵니다.
2. 입력 변수 X와 정답 라벨 y를 분리합니다.
3. 학습 데이터와 테스트 데이터를 나눕니다.
4. 로지스틱 회귀 모델을 학습합니다.
5. 테스트 데이터로 예측을 수행합니다.
6. 정확도, 혼동행렬, 분류 리포트를 확인합니다.
7. 학습된 모델을 models/logistic_model.pkl 파일로 저장합니다.

저장된 모델은 Flask 웹앱에서 불러와 사용자의 입력값에 대한 예측을 수행합니다.

## 8. <> 결과 해석
모델은 사용자의 입력값을 바탕으로 고립은둔 청년일 가능성을 예측합니다.

웹 화면에서는 모델의 예측 확률을 기준으로 고립 위험도를 다음과 같이 구분합니다.

고립은둔 가능성 확률	화면 표시
30% 미만	낮음
30% 이상 60% 미만	보통
60% 이상	높음

1차 모델 평가 결과 전체 정확도는 약 89.36%로 나타났습니다.

혼동행렬은 다음과 같습니다.

[[ 65   9]
 [105 892]]

분류 리포트 주요 결과는 다음과 같습니다.

클래스	의미	Precision	Recall	F1-score
1	고립은둔 청년	0.38	0.88	0.53
2	미해당	0.99	0.89	0.94

고립은둔 청년 클래스의 재현율은 0.88로 비교적 높게 나타났습니다. 이는 실제 고립은둔 청년을 놓치지 않는 방향으로 예측하는 경향을 의미합니다.

반면 고립은둔 청년 클래스의 정밀도는 0.38로 낮게 나타났습니다. 이는 모델이 실제로는 미해당인 응답자 일부를 고립은둔 청년으로 예측하는 오탐이 존재한다는 의미입니다.

본 프로젝트는 고립 위험 가능성을 조기에 참고하기 위한 학습용 예측 웹앱이므로, 단순 정확도뿐만 아니라 고립은둔 청년 클래스의 재현율도 함께 확인했습니다.

## 9. <> 한계점
본 프로젝트에는 다음과 같은 한계가 있습니다.

1. 예측 결과는 진단이 아닙니다.
   본 프로젝트는 공공데이터 기반 학습용 예측 웹앱이며, 실제 심리적·의학적 진단을 대체하지 않습니다.
2. 입력 변수 선정에 한계가 있습니다.
   전체 설문 문항 중 일부 변수만 선택하여 사용했기 때문에, 고립은둔 여부에 영향을 줄 수 있는 모든 요인을 반영하지는 못했습니다.
3. 현재는 결측치가 포함된 행을 제거하는 방식으로 처리했습니다.
    향후에는 평균값, 중앙값, 최빈값 대체 등 다른 결측치 처리 방식과 비교할 수 있습니다.
4. 클래스 불균형이 존재합니다.
   고립은둔 청년 데이터가 미해당 데이터보다 적기 때문에, 모델 성능 해석 시 정확도만으로 판단하기 어렵습니다.
5. 모델 비교가 부족합니다.
   현재는 로지스틱 회귀 모델을 중심으로 구현했으며, 향후 의사결정나무, 랜덤포레스트 등 다른 분류 모델과 성능을 비교할 필요가 있습니다.
6. 서비스 기능이 제한적입니다.
   현재는 입력값 기반 예측 결과를 제공하는 수준이며, 예측 이력 저장, DB 연동, 실제 지원 정보 안내 기능은 포함되어 있지 않습니다.
7. 위험도 구간 기준이 임시적입니다.
   웹 화면에서 사용하는 낮음 / 보통 / 높음 구간은 모델 예측 확률을 바탕으로 설정한 기준이며, 실제 정책적·임상적 기준으로 검증된 것은 아닙니다.
8. 원본 데이터의 조사 범위에 따른 한계가 있습니다.
   본 프로젝트는 서울시 조사 데이터를 기반으로 하므로, 다른 지역이나 다른 연령대에 그대로 일반화하기에는 한계가 있습니다.
